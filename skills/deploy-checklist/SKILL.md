# デプロイチェックリスト: $ARGUMENTS

## デプロイ前チェック

### コード品質
- [ ] `git status` — 未コミットの変更がない
- [ ] CIが全てグリーン (`gh run list --limit 3`)
- [ ] マイグレーションファイルの確認（スキーマ変更がある場合）
- [ ] `.env.example` が最新の環境変数を反映している

### セキュリティ
- [ ] シークレット・APIキーがコードにハードコードされていない
- [ ] 依存パッケージの脆弱性チェック（`npm audit` / `pip-audit` / `govulncheck`）
- [ ] 不要なデバッグログ・コンソール出力の削除

### バックアップ（本番のみ）
- [ ] DBのバックアップが存在する・または自動バックアップが有効
- [ ] ロールバック手順を確認済み

### ステークホルダー確認（本番のみ）
- [ ] デプロイ時間を関係者に通知済み
- [ ] メンテナンスが必要な場合はメンテナンスページを設定済み

---

## デプロイ手順

```bash
# 1. CIのステータス確認
gh run list --limit 5

# 2. deploy-manager エージェントに依頼
# @deploy-manager [staging/production]へのデプロイをお願いします
```

---

## デプロイ後チェック

### 動作確認
- [ ] ヘルスチェックエンドポイントが 200 を返す
  ```bash
  curl -f https://[DOMAIN]/health
  ```
- [ ] 主要機能が正常に動作する（スモークテスト）
- [ ] エラーログにスパイクがない（直後5分間モニタリング）

### パフォーマンス
- [ ] レスポンスタイムが正常範囲内
- [ ] DBコネクションが正常

### ロールバック判断基準
以下のいずれかに該当する場合は即座にロールバック:
- エラー率が通常の5倍以上
- ヘルスチェックが失敗
- 主要機能が動作しない
- DBへの接続ができない

```bash
# ロールバック実行（deploy-manager に依頼）
# @deploy-manager 直前のバージョンにロールバックしてください
```
